## what is mirror.

mirror. is a native desktop IDE that lets designers and developers work on the same project, in the same tool, at the same time — without either side losing their footing. design visually in the canvas. edit directly in code. watch them stay in sync in real time. no exporting, no handoff, no translation. just **two lenses looking at the same truth.**

> "designers shouldn't have to leave their world to understand code. developers shouldn't have to leave their world to understand design. mirror. is the place where neither has to."

---

## why mirror. exists

the gap between design and code costs teams time, causes miscommunication, and makes the creative process feel more fragmented than it needs to be. designers work in one tool. developers work in another. something always gets lost in translation.

mirror. exists to close that gap — completely, natively, and for free. forever.

---

## core features

### bidirectional sync
design and code update each other in real time. change a color in the canvas, it reflects in the CSS instantly. edit the code, the canvas updates. no manual export, no copy-pasting values.

### visual canvas
a fully native design canvas built from scratch. place elements, adjust properties, build layouts — no browser, no wrapper underneath. just your work on screen.

### native code editor
a hand-built code editor with syntax highlighting, live preview, and full HTML, CSS, and JS support. feels like an IDE because it is one.

### truly native
no Electron. no Tauri. no web wrapper. built in C++ from the ground up. fast startup, small binary, runs close to the metal. mirror. belongs on the desktop.

### visual router
map navigation between pages visually on an infinite canvas. connections you draw compile into real routing logic in your project automatically.

### free forever
MIT licensed. no paywalls, no premium tier, no subscriptions. mirror. is a contribution to the community, not a product.

## philosophy

mirror. is built on the belief that design and code are not opposites. they are two representations of the same thing.

this project was built by one developer, from scratch, because the tool didn't exist and it needed to. every architectural decision was made intentionally. every layer is owned completely. the goal was never to ship fast — it was to ship something real.
